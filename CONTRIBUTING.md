# 如何贡献

感谢你对 Cyber Range 项目的关注！我们欢迎各种形式的贡献。

## 🐛 报告 Bug

如果你发现了 Bug，请创建一个 Issue 并包含以下信息：

1. Bug 描述
2. 复现步骤
3. 期望行为
4. 实际行为
5. 环境信息（Go 版本、Node 版本、操作系统等）

## 💡 提出新功能

如果你有新功能的想法，欢迎创建 Issue 讨论。请描述：

1. 功能的目的和使用场景
2. 可能的实现方案
3. 对现有功能的影响

## 🔧 提交代码

### 开发流程

1. Fork 本仓库
2. 创建你的功能分支 (`git checkout -b feature/amazing-feature`)
3. 提交你的修改 (`git commit -m 'Add amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 创建 Pull Request

### 代码规范

**后端 (Go)**
- 使用 `gofmt` 格式化代码
- 遵循 [Effective Go](https://golang.org/doc/effective_go) 规范
- 添加必要的注释和文档
- 编写单元测试

**前端 (React/TypeScript)**
- 使用 ESLint 检查代码
- 组件使用函数式组件和 Hooks
- 使用 TypeScript 类型

### Commit 规范

使用语义化的 Commit 消息：

```
feat: 添加新功能
fix: 修复 Bug
docs: 更新文档
style: 代码格式调整
refactor: 代码重构
test: 添加测试
chore: 其他修改
```

## 📖 改进文档

文档在 `docs/` 目录下，欢迎修正错误或补充内容。
